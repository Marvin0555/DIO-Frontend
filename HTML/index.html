Overview sobre Web e HTTP

Overview

Descrição: O protocolo HTTP é composto por um Header e um Body. Header: é onde se define qual tipo de requisição será feita, tais como GET, POST, HEAD, PUT, DELETE, OPTIONS.

Requisição GET

Descrição: A requisição GET é usada para recuperar informações de um recurso, como uma página da web, um servidor ou uma API. Os dados são enviados na URL e não no corpo da requisição.

Requisição POST

Descrição: A requisição POST é usada para criar um novo recurso no servidor. Os dados são enviados no corpo da requisição e podem incluir informações sensíveis.

Voltar para o início

Requisição PUT

Descrição: O método PUT é utilizado para atualizar um recurso existente. Ele substitui completamente o recurso com os novos dados enviados no corpo da requisição.

Voltar para o início

Requisição PATCH

Descrição: Diferente do PUT, o método PATCH é usado para atualizar parcialmente um recurso, enviando apenas os campos que precisam ser modificados.

Voltar para o início

Requisição DELETE

Descrição: A requisição DELETE é usada para remover um recurso do servidor. Esse método não deve ser usado para ações reversíveis, pois apaga o recurso permanentemente.

Voltar para o início

Requisição HEAD

Descrição: O método HEAD funciona de forma semelhante ao GET, mas a resposta não contém um corpo, apenas os cabeçalhos. Ele é útil para verificar a existência de um recurso ou obter informações sobre ele sem transferir dados desnecessários.

Voltar para o início

Requisição OPTIONS

Descrição: A requisição OPTIONS é usada para obter informações sobre os métodos HTTP suportados por um servidor para um determinado recurso. Normalmente é usada em conjunto com CORS (Cross-Origin Resource Sharing).

Voltar para o início

Requisição TRACE

Descrição: O método TRACE é usado para diagnosticar conexões HTTP. Ele retorna a mesma solicitação enviada pelo cliente para verificar se os dados estão sendo alterados ou modificados no caminho até o servidor.

Voltar para o início

Requisição CONNECT

Descrição: O método CONNECT é utilizado para estabelecer um túnel de comunicação entre o cliente e um servidor de destino. Ele é comumente usado para conexões seguras via HTTPS, permitindo a criação de um canal criptografado.
